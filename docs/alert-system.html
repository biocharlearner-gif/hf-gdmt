HF GDMT Optimizer · Remote Monitoring

The alert system that watches between visits.

Home-device vitals stream in continuously. Eight deterministic rules decide when a reading means something — early enough to act, specific enough to trust.

Every rule is pure and deterministic — no Date.now, no I/O, the clock is injected. The engine only detects and cites; it never orders, titrates, or prescribes. A clinician accepts each finding before a single FHIR resource is written.

03

The rules that govern firing

Why the engine stays quiet until it should not

Recency gate

Never alert on stale data

Any reading older than 14 days is ignored — a device that stopped reporting can't raise a new live alert.

Suppression

Predictive yields to acute

A trend rule fires only when its hard-threshold counterpart did not — no double-flagging the same decline.

Exclusivity

One heart-rate verdict

Bradycardia and tachycardia are mutually exclusive — a single HR reading resolves to at most one.

Outcome loop

Improved vs. still abnormal

Re-evaluation anchors the clock to the latest reading, so closing a Task reflects the true last value — not wall-clock staleness.
